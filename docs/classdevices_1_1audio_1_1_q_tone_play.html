Detailed Description

template<board::Timer NTIMER, board::PWMPin OUTPUT>
class devices::audio::QTonePlay< NTIMER, OUTPUT >

An optimized surrogate to TonePlay structure.

Contrarily to TonePlay, it does not store Tone (i.e. frequency) but Timer prescaler and counter for the desired frequency. The advantage is that when you construct a QTonePlay from constant tones, each tone wll be converted to timer prescaler and counter at compile-time, hence the generated code is smaller and more efficient; this can be useful when your MCU is limited in data size.
